Overview

This homework has two completed parts in this report. Part 1 fits a 2D neural field to an image using positional encoding and a small MLP. Part 2 fits a NeRF to the provided Lego multi-view dataset using sampled rays, sampled 3D points, a NeRF MLP, and volume rendering.

Part 1: Fit a Neural Field to a 2D Image

I implemented a positional-encoding-based MLP in hw4/code/models.py and the full training pipeline in hw4/code/train_part1.py. The dataloader randomly samples normalized pixel coordinates and RGB targets, the model predicts colors in [0, 1], and optimization uses Adam with MSE loss while PSNR is tracked over training.

Model Architecture and Hyperparameters

  • Input: 2D normalized pixel coordinates (u, v)
  • Positional encoding frequencies: default L = 10
  • Network: 4 linear layers with hidden width 256, ReLU activations, and final Sigmoid
  • Optimizer: Adam
  • Learning rate: 1e-2
  • Batch size: 10000
  • Training iterations: 2000
  • Hyperparameter comparison grid: PE frequencies {2, 10} and widths {64, 256}

Provided Reference Image

The reference image run used the default Part 1 setup and reached a final reconstruction quality of 29.247 dB PSNR.

My own image is IMG_6873.jpg. Since the original resolution is very large, I resized it to a maximum image dimension of 512 inside the training script for a manageable MPS training budget. This run reached 31.337 dB PSNR.

Part 2: Fit a Neural Radiance Field from Multi-view Images

I implemented the missing ray sampling pipeline in hw4/code/dataset_3d.py, point sampling along rays and rendering helpers in hw4/code/rendering.py, and the NeRF model plus full training pipeline in hw4/code/models.py and hw4/code/train_nerf.py.

Implementation Summary

  • Part 2.1, create rays from cameras: I used the intrinsics matrix K and each camera-to-world matrix to convert every pixel center into a ray origin and a normalized ray direction.
  • Part 2.2, sample points along rays: I uniformly sampled 3D points between near = 2.0 and far = 6.0 for each ray and added perturbation during training for stratified sampling.
  • Part 2.3, put dataloading together: I implemented a fully working RaysData dataset that precomputes pixel coordinates, ray origins, ray directions, and ground-truth RGB values across all training images, then randomly samples rays from this flattened pool.
  • Part 2.4, neural radiance field: I implemented an 8-layer NeRF MLP with hidden width 256, positional encoding for both 3D points and viewing directions, and a skip connection at layer 4.
  • Part 2.5, volume rendering: I predicted per-sample color and density and composited them with the provided discrete volume rendering equation to obtain one RGB prediction per ray.
  • Stability fix: I changed the density head from a plain ReLU output to Softplus with a small positive bias initialization to prevent the density branch from collapsing to zero during long MPS runs.

Best Training Configuration

  • Dataset: provided lego_200x200.npz
  • Device: mps
  • Training iterations: 2500
  • Rays per iteration: 4096
  • Samples per ray: 32
  • Learning rate: 5e-4
  • XYZ positional encoding frequencies: 10
  • Direction positional encoding frequencies: 4
  • Validation cadence: every 100 iterations on 6 validation views
  • Final validation PSNR: 23.103 dB
